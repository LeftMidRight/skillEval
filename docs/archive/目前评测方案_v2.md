# 金融财报 PDF LAS 解析 Skill — 目前评测方案

> 版本 v2.0 | 2026-06-01

---

## 一、评测目标

评测火山引擎 LAS `las_pdf_parse_doubao` v1 算子对**上证上市公司年报 PDF** 的解析质量。

核心命题：**PDF 解析结果能否支撑下游财务分析任务（事实抽取、指标计算、逻辑推理）？**

## 三、评测框架

### 总分模型

```
总分 = 加权平均（1~10 分制，保留 1 位小数）

模块 1  内容还原度        30%
模块 2  结构组织度        20%
模块 3  鲁棒性与边界      25%
模块 4  端到端业务可用性  25%

独立验证  FinAR-Bench 下游评测
基础章节  行业调研（第零章）
```

### 各模块定义

| 模块 | 核心问题 | 评测维度 |
|------|---------|---------|
| 1 内容还原度 | PDF 里的东西有没有丢、有没有错 | 文本准确性、表格还原度、数值完整性 |
| 2 结构组织度 | PDF 的物理页面结构是否被转换为文档的逻辑结构 | 页级去重、表级拼接、节级完整 |
| 3 鲁棒性与边界 | 异常情况/极端输入下是否 gracefully 处理 | 异常文件、极端排版、语言支持 |
| 4 业务可用性 | 解析结果能否支撑实际财务分析任务 | 下游任务准确率、场景衰减 |

## 四、模块 1：内容还原度（Content Fidelity）

> **核心问题**：PDF 里的东西有没有丢、有没有错？

### 4.1 子指标 1.1 — 文本准确率（CER）

**定义**：Character Error Rate，字符级编辑距离。

$$\text{CER}(\text{ref}, \text{hyp}) = \frac{\text{插入} + \text{删除} + \text{替换}}{|\text{ref}|}$$

**参照系**：
- 主参照：Mineru（公认最优开源解析器）
- 辅助参照：pdfplumber / pymupdf / pypdf / pdfminer / pdftotext
- 输出：CER vs 每个解析器 + 中位 CER（稳健估计）

**实现方式**：
- 短文本（<3000 字符）：精确 Levenshtein 动态规划
- 长文本：difflib 块级对齐 + 非匹配块内精确 CER

**1-10 分映射**（财报场景调整）：

| CER | 分数 |
|-----|------|
| ≤ 0.03 | 10 |
| ≤ 0.06 | 9 |
| ≤ 0.10 | 8 |
| ≤ 0.15 | 7 |
| ≤ 0.22 | 6 |
| ≤ 0.30 | 5 |
| ≤ 0.40 | 4 |
| ≤ 0.55 | 3 |
| ≤ 0.70 | 2 |
| > 0.70 | 1 |

### 4.2 子指标 1.2 — 表格还原度（TEDS + Cell F1）

**TEDS**（Tree Edit Distance based on Structure）：比较 LAS HTML 表格树与 XBRL 表格树之间的结构差异。

- 行匹配：第一列项目名别名匹配（"应收账款"↔"应收帐款"）+ fuzzy match（阈值 0.55）
- 编辑操作：删除行→代价=该行单元格数；替换单元格→代价=1-内容相似度
- **只罚 XBRL 中缺失的项目，不罚 LAS 多出的内容**

**Cell F1**：XBRL 为 gold standard，只测 XBRL 中的项目是否被正确识别。

$$\text{Cell F1} = \frac{\text{TP}}{\text{XBRL 总单元格数}}$$

**表格综合分**：

$$\text{表格分} = 0.6 \times \text{TEDS分数} + 0.4 \times \text{CellF1分数}$$

> TEDS 和 Cell F1 先各自转 1-10 分再加权，而非对原始值直接加权。

**1-10 分映射**（TEDS 和 Cell F1 共用）：

| 指标值 | 分数 |
|--------|------|
| ≥ 0.98 | 10 |
| ≥ 0.95 | 9 |
| ≥ 0.90 | 8 |
| ≥ 0.85 | 7 |
| ≥ 0.75 | 6 |
| ≥ 0.65 | 5 |
| ≥ 0.55 | 4 |
| ≥ 0.40 | 3 |
| ≥ 0.20 | 2 |
| < 0.20 | 1 |

### 4.3 子指标 1.3 — 数值匹配率

**定义**：从 LAS 表格提取的数值与 XBRL 表格数值的集合匹配度。

$$\text{Precision} = \frac{|\text{LAS表数} \cap \text{XBRL表数}|}{|\text{LAS表数}|}
\quad
\text{Recall} = \frac{|\text{LAS表数} \cap \text{XBRL表数}|}{|\text{XBRL表数}|}$$

- 只在表格内提取（避免页码、年份等文本数字污染）
- 标准化：去逗号、括号负数→负号、统一 2 位小数

**1-10 分映射**（按 F1）：

| F1 | 分数 |
|----|------|
| ≥ 0.99 | 10 |
| ≥ 0.97 | 9 |
| ≥ 0.93 | 8 |
| ≥ 0.88 | 7 |
| ≥ 0.80 | 6 |
| ≥ 0.70 | 5 |
| ≥ 0.60 | 4 |
| ≥ 0.45 | 3 |
| ≥ 0.30 | 2 |
| < 0.30 | 1 |

### 4.4 模块 1 总分与结果

$$\text{模块1} = 0.30 \times \text{文本} + 0.40 \times \text{表格} + 0.30 \times \text{数值}$$

**批量结果（25 家公司）**：

| 维度 | 平均分 | 范围 |
|------|--------|------|
| 文本准确率 | 5.9 | 2–8 |
| 表格还原度 | 2.9 | 2–3 |
| 数值匹配率 | 3.9 | 3–5 |
| **模块 1 总分** | **4.1** | **2.6–4.8** |

> 分数集中在 3-5 分区间（24/25 家），评测框架稳定。

## 五、模块 2：结构组织度（Structure Quality）

> **核心问题**：PDF 的物理页面结构（页眉、页码、跨页断裂）是否被转换为文档的逻辑结构？

### 5.1 设计理念

PDF 是一页一页的物理页面，但一份财报是一篇连续的文档。好的解析器应该完成**物理 → 逻辑**的转换：

```
PDF 物理结构                         期望的逻辑输出
┌──────────────────┐
│ 2023 年年度报告   │ ← 页眉            ## 二、财务报表
│ 流动资产合计...   │
│ 71 / 205         │ ← 页码            ### 合并资产负债表
└──────────────────┘
┌──────────────────┐                   | 项目 | 2023 | 2022 |
│ 2023 年年度报告   │ ← 同上            | 流动资产合计 | ... |
│ 长期待摊费用...   │                   | 长期待摊费用 | ... |
│ 公司负责人：xxx   │ ← 签名            | 资产总计 | ... |
│ 72 / 205         │ ← 页码
└──────────────────┘
```

提取页码和页眉是 LAS 的正确行为（说明它没漏内容），但一个完成"物理→逻辑"转换的解析器应该把这些页面装饰合并/去重/移除，输出连续的文档结构。

### 5.2 子指标 2.1 — 页面装饰密度

**测什么**：物理页面残余（页码、重复页眉、签名栏）在输出的占比。

**检测规矩**：
- 页码：`^\d+ / \d+$` 格式（如 `71 / 205`）
- 重复行：全文中出现 ≥3 次的非空行（如 `2023 年年度报告` 出现 8 次）
- 签名栏关键词：含"公司负责人/主管会计/会计机构"等

$$\text{页面装饰密度} = \frac{\text{装饰字符数}}{\text{总字符数}}$$

**1-10 分映射**：≤0.005→10, ≤0.01→9, ≤0.02→8, ≤0.03→7, ≤0.05→6, ≤0.08→5, ≤0.12→4, ≤0.18→3, ≤0.25→2, >0.25→1

> 注意：这测的不是"LAS 做错了"，而是"物理→逻辑转换的残留程度"。

### 5.3 子指标 2.2 — 跨页表格完整性

**测什么**：跨页表格是否被拼回连续整体（而非中间夹着页码/页眉的多个碎片）。

**检测方法**：在 LAS 输出的 markdown 中查找相邻的 `</table>` ... `<table>` 片段，检查它们之间是否有页面标记（页码/页眉）插入。

$$\text{跨页断裂率} = \frac{\text{被页面标记打断的表格片段连接数}}{\text{总表格片段连接数}}$$

**1-10 分映射**：0 个断裂→10 分，每多 1 个断裂扣 1 分，最低 1 分。

### 5.4 子指标 2.3 — 关键节段覆盖

**测什么**：输出中是否包含全部关键财报章节。

| 章节 | 是否必须 |
|------|---------|
| 资产负债表 | ✅ 必须 |
| 利润表 | ✅ 必须 |
| 现金流量表 | ✅ 必须 |

检测通道：
1. Markdown 标题行（`### 合并资产负债表`）
2. 纯文本表名行（无 `#` 标记但含关键词）

**1-10 分映射**：三个必须全有→8 分，含审计报告→+2 分（满分 10），缺一个必须→-3 分，最低 1 分。

### 5.5 模块 2 总分

$$\text{模块2} = 0.40 \times \text{页面装饰密度} + 0.30 \times \text{跨页表格完整性} + 0.30 \times \text{关键节段覆盖}$$

### 5.6 当前结果（25 家公司）

| 维度 | 平均分 |
|------|--------|
| 页面装饰密度 | 7.7/10 |
| 关键节段覆盖 | 8.1/10 |
| 跨页表格完整性 | —（待实现） |
| **模块 2 当前分** | **6.8/10** |

> 25 家公司全部检测到三大报表。噪声密度约 1.4%，主要是页码和重复页眉。

## 六、模块 3：鲁棒性与边界（Robustness & Edge Cases）

> **核心问题**：异常情况/极端输入下是否 gracefully 处理？

### 6.1 子指标 3.1 — 异常文件处理

测试输入：损坏 PDF、加密 PDF、空文件、非 PDF 文件（`.txt` 改名）。

| 行为 | 分数 |
|------|------|
| 返回明确错误码/错误信息 | 7-8 |
| crash / 无响应 | 1 |
| 正常解析（如空文件返回空） | 10 |

### 6.2 子指标 3.2 — A~G 场景专项

将模块 1+2 的指标按 A~G 场景分组重算，检测 LAS 在哪些场景下掉分最严重。

### 6.3 子指标 3.3 — 语言支持

中英混排、纯英文、繁体中文年报的识别率对比。

## 七、模块 4：端到端业务可用性（End-to-End Utility）

> **核心问题**：解析结果能不能支撑实际财务分析？

### 7.1 子指标 4.1 — 下游任务准确率（LLM-as-Judge）

**方法**：
1. 以 LAS 输出 markdown 作为 LLM context
2. 用 FinAR-Bench 的 13 个任务（6 fact + 6 indicator + 1 reasoning）逐题提问
3. LLM 答案 vs XBRL ground truth 对比

### 7.2 子指标 4.2 — 场景衰减分析

按 A~G 场景分组，计算各场景下的下游任务准确率。
