评测火山引擎 LAS `las_pdf_parse_doubao` 算子对**上证上市公司年报 PDF** 的解析质量，为后续财务分析任务（事实抽取、指标计算、逻辑推理）提供可靠的输入。

# 模块 1：内容还原度（Content Fidelity）

> 评测 PDF 里的信息解析出来有没有丢、有没有错。

---

## 1.1 指标体系

| 子指标 | 指标名 | 计算方式 | 参照系 |
|--------|--------|---------|--------|
| 1.1 文本准确率 | CER（Character Error Rate） | Levenshtein 编辑距离 | Mineru / 6 解析器交叉对比 |
| 1.2 表格还原度 | TEDS + Cell F1 | 树编辑距离 + 单元格精确匹配 | XBRL 利润表/资产负债表/现金流量表 |
| 1.3 数值匹配率 | Precision / Recall / F1 | 数值标准化后集合交并比 | XBRL 表格数值 |

---

## 1.2 子指标 1.1 —— 文本准确率（CER）

### 定义

$$
\text{CER}(\text{ref}, \text{hyp}) = \frac{\text{插入} + \text{删除} + \text{替换}}{|\text{ref}|}
$$

### 计算方式

1. 文本清洗：全角→半角，统一空白字符
2. 短文本（<3000 字符）：精确 Levenshtein 动态规划 O(n×m)
3. 长文本：difflib 块级对齐，在非匹配块内做精确 CER，加权平均

### 参照系

- 主参照：Mineru（公认最优解析器）
- 辅助参照：pdfplumber / pymupdf / pypdf / pdfminer / pdftotext
- 输出：CER vs 每个解析器 + 中位 CER（稳健估计）

### 示例验证结果（600064 南京高科）

| 对比 | CER |
|------|-----|
| LAS vs Mineru | 0.175 |
| LAS vs pdfplumber | 0.748 |
| LAS vs pymupdf | 0.750 |
| LAS vs pypdf | 0.750 |
| LAS vs pdfminer | 0.782 |
| LAS vs pdftotext | 0.782 |
| **中位 CER** | **0.750** |

> 解读：LAS 与 Mineru 最接近，与其他解析器差距较大，说明 LAS 输出风格更接近 Mineru 的高保真路线。

### 1-10 分映射（财报场景调整版）

| CER | 分数 | 说明 |
|-----|------|------|
| ≤ 0.03 | 10 | 几乎完全一致 |
| ≤ 0.06 | 9 | 优秀 |
| ≤ 0.10 | 8 | 良好 |
| ≤ 0.15 | 7 | 中等偏上 |
| ≤ 0.22 | 6 | 中等 |
| ≤ 0.30 | 5 | 一般 |
| ≤ 0.40 | 4 | 较差 |
| ≤ 0.55 | 3 | 差 |
| ≤ 0.70 | 2 | 很差 |
| > 0.70 | 1 | 几乎不一致 |

## 1.3 子指标 1.2 —— 表格还原度（TEDS + Cell F1）

### TEDS（Tree Edit Distance based on Structure）

比较 LAS HTML 表格树与 XBRL 表格树之间的结构差异。

**编辑操作**：
- 删除行（XBRL 有的行 LAS 没有）→ 代价 = 该行单元格数
- 插入行（LAS 有多余行不在 XBRL 中）→ **不罚分**（XBRL 是精简版 gold standard）
- 单元格替换 → 代价 = `1 - content_similarity(cell1, cell2)`

**行匹配**：用第一列项目名做别名匹配（"应收账款"↔"应收帐款"）+ fuzzy match（阈值 0.55）。

**计算**：
$$
\text{TEDS} = \max\left(0,\ 1 - \frac{\text{编辑距离}}{\max(|T_1|, |T_2|)}\right)
$$

### Cell F1

**策略**：XBRL 为 gold standard，只测 XBRL 中的项目是否在 LAS 中被正确识别。
不罚 LAS 多提取的内容。

$$
\text{Precision} = \text{Recall} = \frac{\text{TP}}{\text{XBRL 总单元格数}}
$$

**数值匹配**：允许 ±0.1% 相对误差（容忍 PDF 最小精度位差异）。

### 表格综合分

将原始 TEDS 和 Cell F1 各自转为 1-10 分，再加权：

$$
\text{表格分} = \text{round}(0.6 \times \text{TEDS分数} + 0.4 \times \text{CellF1分数})
$$

### 单元验证

| 测试用例 | TEDS | 预期 |
|---------|------|------|
| 完全相同的两表 | 1.000 | ✓ |
| 缺一行（3/9 cells） | 0.667 | ✓ |
| 一个单元格值不同 | 0.889 | ✓ |
| colspan 差异 | 0.942 | ✓ |
| XBRL 自检（XBRL = XBRL） | Cell F1 = 1.000 | ✓ |

### 示例验证结果（600064）

| 报表 | Cell F1 | TEDS | 说明 |
|------|---------|------|------|
| 利润表 | 0.364 | 0.488 | LAS 行数远多于 XBRL，结构差异大 |
| 资产负债表 | 0.437 | 0.504 | 最好——行匹配较多 |
| 现金流量表 | 0.000 | 0.353 | 匹配到"母公司现金流量表"而非"合并现金流量表" |
| **整体** | **0.289** | **0.479** | |

## 1.4 子指标 1.3 —— 数值匹配率

### 计算方式

1. 从 XBRL 三张表提取所有数值单元格 → 标准化（去逗号、括号负数→负号、统一 2 位小数）
2. 从 LAS 输出表格中提取所有数值单元格 → 同样标准化
3. 取集合交集

$$
\text{Precision} = \frac{|\text{LAS数} \cap \text{XBRL数}|}{|\text{LAS数}|}
\quad
\text{Recall} = \frac{|\text{LAS数} \cap \text{XBRL数}|}{|\text{XBRL数}|}
$$

### 关键设计决策

- **只从表格提取**：不用正则从全文提取（避免页码、年份、正文数字污染）
- **标准化到 2 位小数**：容忍 PDF 精度丢位
- **百分号剥离**："12.34%" → 12.34

### 示例验证结果（600064）

| 指标 | 值 |
|------|-----|
| Recall | 98.3%（177/180 个 XBRL 数值被找到） |
| Precision | 39.8%（LAS 表格有 445 个数值，268 个不在 XBRL 中） |
| **F1** | **0.566** → 得分 4/10 |

> 解读：Recall 接近完美，说明 LAS 几乎不漏；Precision 偏低是因为 LAS 解析了完整报表（含附注），XBRL 只含核心项目。

### 1-10 分映射

| F1 | 分数 |
|----|------|
| ≥ 0.99 | 10 |
| ≥ 0.97 | 9 |
| ≥ 0.93 | 8 |
| ≥ 0.88 | 7 |
| ≥ 0.80 | 6 |
| ≥ 0.70 | 5 |
| ≥ 0.60 | 4 |
| ≥ 0.45 | 3 |
| ≥ 0.30 | 2 |
| < 0.30 | 1 |

## 1.5 模块 1 总分

$$
\text{模块1} = 0.30 \times \text{文本} + 0.40 \times \text{表格} + 0.30 \times \text{数值}
$$

表格权重最高（40%），因为它是最核心的财报信息载体，且是唯一有 XBRL gold standard 的维度。

## 1.7 已知局限与改进方向

| 问题 | 影响 | 改进方向 |
|------|------|---------|
| XBRL 是精简版（关键项目），LAS 是完整版（全文） | 表格 Precision 偏低 | 单独构建一个完整项目列表做参照 |
| 现金流量表匹配不稳 | 个别公司 F1=0 | 增加"母公司/合并"的灵活标题匹配 |
| HTML colspan 被 LAS 丢失 | TEDS 偏低 | 评测侧已适配，修复需 LAS 侧 |
| CER 只有 Mineru 做参照 | 无绝对 ground truth | 多解析器交叉对比矩阵已经有，足够稳健 |
